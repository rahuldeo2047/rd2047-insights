Act 0  ·  The Problem

Every automated pipeline has a silent failure mode

API credit exhaustion. It happens mid-cycle, without warning. The queue goes empty. Scheduled content doesn't go out. The system is healthy — it just has no brain.


The engineering objective: build a local AI inference node that generates structured content autonomously — offline, no rate limits, no billing — and activates automatically when cloud services are unavailable. The output schema had to be byte-compatible with the cloud path. The downstream renderer would never know which tier ran.

The Hard Constraint
Format compliance wasn't a nice-to-have. It was the entire problem. Slides that couldn't be parsed by the renderer were worse than no slides. Every benchmark iteration measured one thing: can this model produce output a regex grader accepts 100% of the time?
Act 1  ·  Hardware

The Stack

The Raspberry Pi AI HAT+ 2 pairs an RPi5 with a Hailo-10H NPU — 40 TOPS INT4, 8 GB dedicated on-board DRAM. The H10 handles all inference. The Pi 5 CPU stays near-zero. Models are pre-compiled to HEF (Hailo Executable Format) — no GGUF, no llama.cpp, no CUDA.

Critical Discovery — Before Writing Any Code
hailo-ollama is not standard Ollama. It implements only the REST side. Port 11434 doesn't exist. The ollama CLI binary is incompatible. format:"json", stop sequences, and assistant prefill are silently ignored. Every assumption about Ollama compatibility needed empirical verification before a benchmark was written.

Act 3  ·  The Benchmark

Fifteen iterations to convergence

A 4-rule heuristic grader scored every model output: exact CTA string match (25 pts), SCRIPT word count ≤ 8 (25 pts), numerical data present (25 pts), forbidden title words absent (25 pts). Five models tested across 15 iterations of prompt engineering.

The Grader Bug — Present in Iterations 6 through 13
The CTA normalisation regex kept @ symbols in the text but the target string had no @ — so "follow @rahuldeo2047" never matched "follow rahuldeo2047". Every model that produced the correct CTA string was scored 0 for seven iterations. DeepSeek-R1 was at 100% from iteration 6 — the grader was wrong, not the model. This is why you verify failure cases manually before trusting metrics.

What the iterations taught
Format examples in system prompts must use a domain unrelated to the test niche — otherwise the model echoes examples into real output. num_predict must be model-specific not slide-specific — DeepSeek needs 800 tokens for its reasoning chain; 60 kills it entirely. FAIL examples actively harm small models — they copy them verbatim.

Act 6  ·  Production Integration

The fallback chain

The local node slots into the pipeline as tier 3. All tiers produce the identical slide schema — the downstream renderer never knows which tier ran. Every slide dict carries generator and model_version for future analytics correlation.

Cloud Primary
Claude via Anthropic API. Full analytics context. Best quality. Runs when credits available.
